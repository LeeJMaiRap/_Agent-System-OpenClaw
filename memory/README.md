# memory

Workspace memory folder for continuity notes, operational audit records, migration/rollback context, and compact summaries of long-running work.

## Current contents

- Daily notes: `YYYY-MM-DD.md`.
- Migration and rollback notes: migration checklists, rollback source, old-workspace scan summaries.
- Agent-System audit artifacts: path audits, broken links, profile/skills verification JSON, README inventories/review reports.
- Semantic README review artifacts.
- `.dreams/` recall files generated by OpenClaw runtime.

## Use for

- Human-readable continuity and decisions.
- Operational audit evidence.
- Migration/rollback context.
- Durable summaries that help future sessions understand workspace history.

## Do not use for

- Secrets, tokens, credentials, private keys.
- Build/cache/dependency outputs such as `node_modules/`, `.venv/`, `dist/`, `build/`.
- Noisy transient raw dumps unless explicitly needed for audit/rollback and summarized.

## Current preference remembered

For long command/tool output, default to compact output: keep key points, paths, counts, and decisions; save full raw output to a file only when useful; offer pagination only on request.
