# Agent System

Agent-System is the active workspace area for agent profiles, adapted skills, documentation, schemas, reports, validation tests, and utility tools.

## Source of truth

- Active PM Agent: `agents/software/pm-agent/`.
- Legacy PM adapter: `agents/software/pm-agent-adapter-legacy/` (historical only).
- Shared handoff protocol: `docs/specialist-handoff-protocol.md`.
- Task templates: `templates/specialist-task-packet.md`, `templates/specialist-task-report.md`.

## Areas

- `agents/` — 11 agent profiles across software and business/affiliate domains.
- `skills-library/` — adapted software and affiliate skills used by profiles.
- `docs/` — activation guide, role matrix, roadmap, safety policy, handoff protocol, schema notes.
- `audits/` — source/repo/skill audit notes.
- `tests/` — Paper-level and read-only validation artifacts.
- `reports/` — hardening/schema/consistency reports.
- `schemas/` — JSON schema for agent skills metadata.
- `templates/` — specialist packet/report templates.
- `tools/` — utility scripts: `check-consistency.py` (Python 3) and `check-consistency.js` (Node fallback).
- `sources/` — source-material staging area; currently present but not populated with active source files.

## Current status

Profiles and skills are draft/Paper-validated assets unless specific evidence says otherwise. External actions, posting, purchases, ads, payments, API keys, secrets, destructive actions, and production deployment require explicit human approval.
